02. RECOMMENDED LOG READING METHOD
============================================================

Recommended tools:
  - CMTrace
  - OneTrace
  - Support Center Log File Viewer

General troubleshooting order:
  1. Check the client-side log first if the issue affects one device.
  2. Check the server-side log if the issue affects many devices.
  3. Search by:
       - Application name
       - Package ID
       - Update KB number
       - Assignment ID
       - Deployment ID
       - Error code
       - Device name
  4. Follow the SCCM flow:
       Policy received
       Evaluation / detection
       Content location
       Download
       Installation
       State message / reporting
  5. Always check date/time difference between SCCM server and client.
  6. Check boundary group if client cannot download content.
  7. Do not keep verbose/debug logging enabled permanently.

03. CLIENT AGENT INSTALLATION / REPAIR
============================================================

Use this section when:
  - SCCM client is not installing
  - Client shows "No" in console
  - Client is inactive
  - Configuration Manager applet is missing
  - Software Center is missing
  - Client push installation fails

Important logs:

  ccmsetup.log
    Location: C:\Windows\ccmsetup\Logs
    Purpose : Main SCCM client installation, repair, upgrade, uninstall log.

  client.msi.log
    Location: C:\Windows\ccmsetup\Logs
    Purpose : MSI-level installation details.

  ccmsetup-ccmeval.log
    Location: C:\Windows\ccmsetup\Logs
    Purpose : Client health evaluation during setup.

  CcmRepair.log
    Location: C:\Windows\CCM\Logs
    Purpose : SCCM client repair activity.

  CcmEval.log
    Location: C:\Windows\CCM\Logs
    Purpose : Client health evaluation result.

  CcmEvalTask.log
    Location: C:\Windows\CCM\Logs
    Purpose : Scheduled client evaluation task details.

  ClientIDManagerStartup.log
    Location: C:\Windows\CCM\Logs
    Purpose : Client GUID, certificate, registration, and identity.

  CcmExec.log
    Location: C:\Windows\CCM\Logs
    Purpose : SMS Agent Host service activity.

  LocationServices.log
    Location: C:\Windows\CCM\Logs
    Purpose : Finds Management Point, Distribution Point, SUP, site assignment.

  ClientLocation.log
    Location: C:\Windows\CCM\Logs
    Purpose : Client site assignment and location.

  ccm.log
    Location: Site Server Logs
    Purpose : Server-side client push installation activity.

Troubleshooting flow:
  Client push failed:
    1. Check ccm.log on site server.
    2. Check admin$ access, firewall, WMI, RPC.
    3. Check ccmsetup.log on client.

  Client installed but inactive:
    1. Check CcmExec.log.
    2. Check ClientIDManagerStartup.log.
    3. Check LocationServices.log.
    4. Check CcmMessaging.log.

  Client cannot find site:
    1. Check ClientLocation.log.
    2. Check LocationServices.log.
    3. Check boundary and boundary group.

05. SOFTWARE CENTER
============================================================

Use this section when:
  - Software Center does not open
  - Applications are not showing
  - Updates are not showing
  - Install button missing
  - Software Center shows wrong status

Important logs:

  SCClient_domain@username_1.log
    Purpose: Software Center user interface activity.

  SCClient_domain@username_2.log
    Purpose: Additional Software Center UI activity.

  SoftwareCenterSystemTasks.log
    Purpose: Software Center background tasks.

  PolicyAgent.log
    Purpose: Checks if deployments are received.

  AppIntentEval.log
    Purpose: Application applicability and requirement evaluation.

  AppDiscovery.log
    Purpose: Application detection.

  UpdatesDeployment.log
    Purpose: Software update deployment evaluation.

Troubleshooting flow:
  App not showing in Software Center:
    1. Check if device/user is in target collection.
    2. Check deployment purpose: Available or Required.
    3. Check User Experience visibility.
    4. Run Machine Policy Retrieval.
    5. Check PolicyAgent.log.
    6. Check AppIntentEval.log.

    1. Check if update is Required for the client.
    2. Check deployment target collection.
    3. Check Scheduling / Available Time.
    4. Check User Experience visibility.
    5. Run Software Updates Scan Cycle.
    6. Run Software Updates Deployment Evaluation Cycle.
    7. Check UpdatesDeployment.log and UpdatesStore.log.

07. SOFTWARE UPDATE DEPLOYMENT
============================================================

Use this section when:
  - Windows update does not show in Software Center
  - Update is Required but not visible
  - Update stuck downloading
  - Update stuck installing
  - Update shows Unknown in console
  - Compliance status is wrong
  - Update deployment not evaluated

Important logs:

  UpdatesDeployment.log
    Purpose: Update deployment activation, evaluation, and enforcement.

  UpdatesHandler.log
    Purpose: Download and installation of updates.

  UpdatesStore.log
    Purpose: Update compliance status: Required, Installed, Not Required.

  WUAHandler.log
    Purpose: Windows Update Agent scan activity.

  ScanAgent.log
    Purpose: Scan request, SUP/WSUS location, scan process.

  PolicyEvaluator.log
    Purpose: Client policy evaluation.

  StateMessage.log
    Purpose: Sends update status/compliance messages to Management Point.

  RebootCoordinator.log
    Purpose: Restart coordination after update installation.

  ServiceWindowManager.log
    Purpose: Maintenance window evaluation.

  CAS.log
    Purpose: Content location and local cache.

  ContentTransferManager.log
    Purpose: Content transfer jobs.

  DataTransferService.log
    Purpose: BITS download activity.

Troubleshooting flow:
  Update not showing:
    1. Confirm client is in target collection.
    2. Confirm update is Required for that client.
    3. Check deployment User Experience.
    4. Check deployment Scheduling.
    5. Run Machine Policy Retrieval.
    6. Run Software Updates Scan Cycle.
    7. Run Software Updates Deployment Evaluation Cycle.
    8. Check UpdatesDeployment.log.
    9. Check UpdatesStore.log.

  Update scan failed:
    1. Check WUAHandler.log.
    2. Check ScanAgent.log.
    3. Check WindowsUpdate.log.
    4. Check SUP/WSUS health.

  Update download failed:
    1. Check CAS.log.
    2. Check ContentTransferManager.log.
    3. Check DataTransferService.log.
    4. Check boundary group and DP.

  Update install failed:
    1. Check UpdatesHandler.log.
    2. Check WUAHandler.log.
    3. Check CBS.log.
    4. Check Windows Event Viewer.

Useful log messages:
  "Assignment is still not active"
    Meaning: Deployment exists, but available time has not reached yet.
    Check : Deployment Scheduling tab and client/server time.

  "Total actionable updates = 0"
    Meaning: No update currently available/actionable for install.
    Check : Required state, schedule, policy, and detection.

  "Update is not required"
    Meaning: Client scan says update does not apply.
    Check : OS version, architecture, installed update status.

08. APPLICATION DEPLOYMENT
============================================================

Use this section when:
  - Application not showing in Software Center
  - Application install fails
  - Uninstall fails
  - Detection method is wrong
  - App installed but Software Center still shows Install
  - Requirement rule fails
  - Dependency/supersedence issue

Important logs:

  AppIntentEval.log
    Purpose: Application requirement, dependency, supersedence, applicability.

  AppDiscovery.log
    Purpose: Application detection method result.

  AppEnforce.log
    Purpose: Actual install/uninstall command execution.

  CAS.log
    Purpose: Content cache and DP location.

  ContentTransferManager.log
    Purpose: Download job creation.

    Purpose: BITS download.

  DCMAgent.log
    Purpose: Application and compliance evaluation.

  CIAgent.log
    Purpose: Configuration item/application compliance process.

  Ccmsdkprovider.log
    Purpose: Application management SDK activity.

Troubleshooting flow:
  Application not showing:
    1. Confirm collection membership.
    2. Confirm deployment is enabled.
    3. Confirm deployment availability time.
    4. Run Machine Policy Retrieval.
    5. Check PolicyAgent.log.
    6. Check AppIntentEval.log.

  Application install failed:
    1. Check AppEnforce.log.
    2. Check command line.
    3. Check installation account/system context.
    4. Check exit code.

  Detection method wrong:
    1. Check AppDiscovery.log.
    2. Confirm detection path/registry/MSI product code.
    3. Correct detection rule.

  Content download failed:
    1. Check CAS.log.
    2. Check ContentTransferManager.log.
    3. Check DataTransferService.log.
    4. Check DP and boundary group.

  - Driver install fails
  - Domain join fails
  - App install fails during OSD
  - Client install fails during OSD

Important logs:

  smsts.log
    Purpose: Main task sequence log.

  TSAgent.log
    Purpose: Task sequence dependency checks before starting.

  CAS.log
    Purpose: Finds DP for task sequence content.

  ccmsetup.log
    Purpose: Client installation during OSD.

  client.msi.log
    Purpose: MSI details for SCCM client installation.

  MP_ClientID.log
    Location: Management Point
    Purpose : MP response to PXE/boot media client identity requests.

  MP_DriverManager.log
    Location: Management Point
    Purpose : Auto Apply Driver requests.

  DriverCatalog.log
    Location: Site Server
    Purpose : Imported driver catalog details.

  OfflineServicingMgr.log
    Location: Site Server
    Purpose : Offline servicing of WIM images.

  Dism.log
    Purpose: Driver/update servicing.

  scanstate.log
    Purpose: USMT capture.

  loadstate.log
    Purpose: USMT restore.

Troubleshooting flow:
  Task sequence failed:
    1. Open smsts.log.
    2. Search "error", "failed", and "return code".
    3. Check the step name.
    4. Check content access if error is download-related.
    5. Check AppEnforce.log if application installation step failed.

  Domain join failed:
    1. Check smsts.log.
    2. Check NetSetup.log.
    3. Check domain account permission.
    4. Check DNS and network.

  Driver issue:
    1. Check smsts.log.
    2. Check MP_DriverManager.log.
    3. Check Dism.log.

    Location: Site Server
    Purpose : Configuration Manager VSS writer.

  ConfigMgrSetup.log
    Location: Site Server
    Purpose : Setup and site recovery tasks.

Troubleshooting flow:
  Site backup failed:
    1. Check smsbkup.log.
    2. Check backup destination permission.
    3. Check SQL backup.
    4. Check disk space.

  Recovery issue:
    1. Check ConfigMgrSetup.log.
    2. Check SQL database restore.
    3. Check site code and prerequisites.

ERROR / MESSAGE:
  Assignment is still not active

MEANING:
  The deployment policy is received, but the deployment available time has not arrived.

CHECK:
  - Deployment Scheduling tab
  - Software available time
  - Client local time
  - Server time
  - Time zone


ERROR / MESSAGE:
  Total actionable updates = 0

MEANING:
  Client has no update currently available for install.

CHECK:
  - Is the update Required?
  - Is deployment active?
  - Is update already installed?
  - Is scan successful?
  - Is policy received?


ERROR / MESSAGE:
  Failed to get targeted update

MEANING:
  Client cannot find or evaluate the targeted update CI correctly.

CHECK:
  - UpdatesDeployment.log
  - UpdatesStore.log
  - WUAHandler.log
  - Software update scan
  - SUP sync status


ERROR / MESSAGE:
  Content location request failed

MEANING:
  Client cannot find valid DP/content location.

CHECK:
  - Boundary
  - Boundary Group
  - DP content status
  - LocationServices.log
  - CAS.log

  Detection method failed

MEANING:
  SCCM cannot confirm whether the application is installed.

CHECK:
  - AppDiscovery.log
  - Detection method rule
  - File path / registry key / MSI product code

  Exit code 1603

MEANING:

CHECK:
  - AppEnforce.log
  - MSI log
  - Installation command
  - Admin rights
  - Prerequisites

  Exit code 3010

MEANING:
